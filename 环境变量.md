## Vercel Active CPU 优化配置

### 超时设置
```bash
# 首包超时（AI 首次响应时间）
FIRST_CHUNK_TIMEOUT_MS=30000

# 总软超时（整个生成过程上限）
TOTAL_SOFT_TIMEOUT_MS=120000

# 心跳间隔（仅在空闲时发送）
HEARTBEAT_INTERVAL_MS=15000

# 去重缓存时间（分钟）
DEDUP_TTL_MINUTES=120

# 每用户最大并发数
MAX_CONCURRENT_PER_USER=1
```

## 配置说明

### Active CPU 优化策略

1. **并发控制**
   - `MAX_CONCURRENT_PER_USER=1`: 每个用户同时只能有一个生成任务
   - 防止用户重复点击导致资源浪费

2. **超时保护**
   - `FIRST_CHUNK_TIMEOUT_MS=30000`: 30秒内必须收到AI首包响应
   - `TOTAL_SOFT_TIMEOUT_MS=120000`: 2分钟内完成整个生成过程
   - 避免长时间占用CPU资源

3. **去重缓存**
   - `DEDUP_TTL_MINUTES=120`: 相同参数的结果缓存2小时
   - 减少重复生成，节省CPU和API调用

4. **Token 限制**
   - 免费用户：1024 tokens（约500-800字）
   - 付费用户：2048 tokens（约1000-1600字）
   - 控制生成长度，减少处理时间

5. **心跳优化**
   - `HEARTBEAT_INTERVAL_MS=15000`: 仅在空闲时发送心跳
   - 保持连接活跃，避免Vercel中断

### 成本优化建议

1. **监控 Active CPU 使用**
   - 访问 `/api/admin/active-cpu-stats` 查看实时统计
   - 关注平均生成时间和并发用户数

2. **调整参数**
   - 根据实际使用情况调整超时时间
   - 优化token限制以平衡质量和成本

3. **缓存策略**
   - 增加去重缓存时间可减少重复生成
   - 但要注意内存使用

4. **并发控制**
   - 降低 `MAX_CONCURRENT_PER_USER` 可减少峰值负载
   - 但可能影响用户体验

## 故障排除

### 常见问题

1. **Active CPU 消耗过高**
   - 检查并发用户数
   - 调整超时参数
   - 优化token限制

2. **生成超时**
   - 增加 `FIRST_CHUNK_TIMEOUT_MS`
   - 检查网络连接
   - 验证API密钥

3. **缓存命中率低**
   - 增加 `DEDUP_TTL_MINUTES`
   - 检查参数哈希算法
   - 分析用户行为模式
